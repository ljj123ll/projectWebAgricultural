## 6. 最安全的主演示链路

如果你希望尽量不改动数据、尽量不出错，建议按下面顺序走：

### 第一阶段：用户端静态展示

1. 登录用户端。
2. 打开首页 `/home`。
3. 打开商品详情 `/product/17`。
4. 打开商家主页 `/shop/8`。
5. 打开独立溯源页 `/trace/TRACE-2038330668139814913`。

这一阶段重点讲：

1. 助农平台首页能力。
2. 商品详情展示。
3. 商家评分与评论聚合。
4. 一物一码和独立溯源页。

### 第二阶段：用户端已有订单展示

1. 打开 `/orders`。
2. 进入一个现有订单详情页 `/order-detail/:orderId`。
3. 展示订单状态、物流、评价和售后入口。

### 第三阶段：商家端经营展示

1. 登录商家端。
2. 打开 `/merchant/product-add`。
3. 展示分类溯源字段和二维码生成逻辑。
4. 打开 `/merchant/orders`。
5. 展示发货流程和防连点。

### 第四阶段：管理员端治理展示

1. 登录管理员端。
2. 打开 `/admin/merchants`。
3. 打开 `/admin/products`。
4. 打开 `/admin/payments`。
5. 打开 `/admin/backup` 或 `/admin/comments`。

## 9. 高风险操作提醒

答辩现场不建议临时尝试以下操作：

1. 随意新增大量测试数据。
2. 对管理员退款状态做多次反复切换。
3. 对商家发货按钮反复点击测试。
4. 在老师面前直接修改数据库。
5. 临时改动后端配置或缓存配置。

## 10. 推荐答辩说法

### 项目一句话介绍

这是一个面向助农场景的三端 Web 电商平台，包含用户端、商家端和管理员端，并重点实现了独立溯源档案、一物一码、平台审核、售后协商和资金监管等能力。

### 建议强调的亮点

1. 不是普通商城，而是助农电商平台。
2. 不是只展示商品，还实现了独立溯源页和二维码直达。
3. 商家端不仅发商品，还维护分类溯源字段。
4. 管理员端不仅看数据，还能做审核、退款、风控、备份。
5. 针对答辩高频风险，项目补了支付防重、取消订单防重、发货防连点和退款状态校验。
