# 2系统简介
督办管理系统，系统名称为“督办管理系统”，是一个Java语言开发的基于Web的系统，系统的主要功能有：用户登录、查询任务、发送通知、审批通知、修改密码、用户的增删改查管理和退出系统。

系统的用户角色有三个，分别为党政办公室（领导）、党政办公室（管理员）、业务单位（普通用户）。

- 开发平台：Webstorm, intelliJ IDEA
- 主要技术：Html, Css, Javascript, Java

## 2.2 系统功能操作说明
用户为普通用户时，登录进入系统，可以进行任务管理，这里的任务分成用户为承办单位时的任务和用户为协办单位时的任务。当用户为协办单位时，可以提交任务的延期申请和任务的回执；

用户为管理员时，登录进入系统，可以发送任务通知，对用户进行增删改查，处理申请延期的任务和查看回执，对进行中的任务可以进行删除和注销；

用户为领导时，登录进入系统，可以审批任务通知和查看任务详情；

所有用户都可以修改自己账号的密码。

督办管理系统的主要业务流程有三种情况，定义为三种场景：
- 场景一：正常场景，领导批准通知，业务单位顺利完成任务。
- 场景二：异常场景，领导不批准通知。
- 场景三：异常场景，业务单位不能按期完成，提出延迟申请。

# 3系统测试说明
对使用Java语言编写的基于Web的督办系统进行了功能测试、兼容性测试和安全测试。功能测试主要进行的是黑盒测试，用以检验系统的各个功能是否完善，是否是按照制定的需求进行设计的，用例的设计方法采用了场景法。功能测试中还对系统进行了链接测试，链接测试主要是对网站的死链接和图片链接进行检测和分析，使用的工具是Xenu link sleuth。

兼容性测试主要是测试浏览器兼容性和分辨率兼容性，即测试系统在不同版本的浏览器上运行，页面和功能上有没有错误或缺陷，同时测试页面版式在各个分辨率模式下页面显示是否正常，使用浏览器中的开发者工具进行测试。

安全测试主要是检查系统自身在设计过程中存在的安全隐患和系统对一些非法侵入的防患能力，本文中的安全测试主要是检查是否存在跨站执行脚本的漏洞和Sql注入漏洞[2]。

# 4测试方法
对督办管理系统的测试，主要是对其进行功能测试、兼容性测试和安全测试。功能测试中首先选择黑盒测试方法对系统进行了测试，测试系统是否按照制定的需求正常运行，测试不同权限的用户在系统中进行的一系列操作是否合乎要求。

# 7 安全性测试
## 7.1 Sql注入测试
### 7.1.1测试流程
因为大多数的登录页面的输入字段都被用作动态Sql语句的一部分，所以本次测试就以登录页面为例，首先假设登录页面有Sql注入漏洞，测试步骤如下：
（1）填写好正确的用户名和密码后提交，提交的用户名和密码被合成到Sql语句中，合成后的Sql语句则应为`SELECT * FROM user WHERE use_name='徐亮' AND use_password= '8888'`；
（2）在用户名输入框中输入`’or 1=1#`,密码输入框任意输入，这时候合成后的Sql查询语句为`SELECT * FROM user WHERE use_name='’or 1=1#' AND use_password='8888'`。

步骤一中，用户名和密码填写正确，Sql语句也应该正常执行。但是在步骤二中，因为“#”在mysql中是注释符，所以它后面的内容会被视为注释内容，就不会被执行。这时的Sql语句`SELECT * FROM user WHERE use_name='’or 1=1#' AND use_password='8888'`就应该等价于`SELECT * FROM user WHERE use_name='’or 1=1`。因为1=1永远成立，所以WHERE子句恒为真，所以`SELECT * FROM user WHERE use_name='’or 1=1`同样等价于`SELECT * FROM user`，即检索user表中所有的数据，如果登录页面有Sql注入漏洞，就能通过输入`’or 1=1#`直接登录成功。

### 7.1.2 测试结果
系统无Sql注入漏洞。

## 7.2 跨站执行脚本测试
### 7.2.1 测试介绍
跨站攻击英文缩写为XSS，是指利用程序对用户的输入值没有进行足够的过滤，输入了可显示在页面上对其它的用户造成影响的代码，从中盗取用户信息等行为的一种攻击方式[8]。本文做的就是在页面的文本输入区域，输入攻击字符串，然后查看页面的响应，如果这段代码被执行出来，那么这个输入字段就存在跨站执行脚本的漏洞。

### 7.2.2 测试流程
因为督办管理系统中大多数输入框都对输入的字符数量进行了限制，所以本文验证的字段为管理员发送任务通知时的“督办事项”和“内容及要求”，这里以对“督办事项”测试跨站执行脚本为例，对“内容及要求”的有效性验证同理。步骤如下：

（2）进入发送通知页面
