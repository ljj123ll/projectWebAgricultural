# 基本概念的概述

## 设备管理的概念

设备管理是指对设备从挑选、使用、维护、修理、更新改造直至报废全过程的管理工作的总称。一般来说，设备在其使用寿命周期内有两种运动的形态；一是物质运动形态，包括设备的选购、进厂验收、安装、调试、使用、维修、改造更新等，对设备的物质运动形态的管理称为设备的技术管理；一是价值运动的形态，包括设备的购进费用、维修费用支出、更新改造资金的支出等，对价值运动形态的管理称为设备的经济管理。

## 系统的概念

中国著名学者钱学森认为：系统是由相互作用相互依赖的若干组成部分结合而成的，具有特定功能的有机整体，而且这个有机整体又是它从属的更大系统的组成部分。[1] 此处我们谈论的是软件系统，软件系统是用于对某些资源进行统一控制、调度和管理，提高工作效率。

# 国内外对设备管理系统的研究现状

对于企业而言，设备管理不仅仅是企业产量、质量的保证，更是效率和交货期的保证[15]。在六十年代末到七十年代初之间，随着发展和市场需要，设备管理的理论和实践有了巨大突破——设备管理的概念不再仅仅局限于维修管理的狭义概念，而是发展为管理整个设备生命周期的广义概念，由此，设备管理成为了影响企业效率的关键组成部分。在企业设备管理理论发展的过程中，相继诞生了《计划预修制》、《设备综合工程学》、《后勤学》[2] 等时代性的设备管理理论。

由于国内当时的封闭思想，我国对于设备管理的思想与技术严重滞后，所以开发的设备管理信息系统基本上停留在对设备的基础信息的统计和管理，而国外对设备管理系统的研究和开发运用比我们早，所以功能机机制也比较完善，形成了具有决等策功能的综合设备管理信息系统，成为了企业现代化生产的重要手段，我国主要以学习国外的先进的管理技术为主。我国的设备管理经历了三个大阶段：经济管理阶段、科学管理阶段、现代化管理阶段。

改革开放之后，部分企业引用了国际上较为先进的设备管理的理论思路，同时探索着适合我国的设备管理方式。1981年4月国务院领导对于加强设备管理作出重要批示，在国家经济委员会内新增设备管理维修办公室，作为全国设备管理工作的统筹办事机构。1982年中国设备管理协会成立，将此作为企业和政府之间加强联系的一个媒介。从这以后设备管理专业机构日趋完善，同时设备管理法规也开始制定，我国设备管理进入了一个健康发展的现代管理阶段。[2]

国外的设备管理系统更注重的是预防和预测、优化成本。以资产、设备相关的费用台账为基础，按设备预防性维修，从而提高维修的效率和成本，降低总成本，将采购管理、库存管理等内容也合为一体，从而实现了数据的共享，也优化库存，提升采购的效率，降低总的运营成本。并且通过预防性的维护，减少了设备的闲置时间，使设备始终处于良好的忙碌的状态，也控制了设备的维修、更新费用，更有效地分配设备资源。

国内的设备管理也在不断发展，在现在的科技时代，充分地运用了现代技术，发展出了基于网络和状态监测的设备管理信息系统。系统将采集的设备运行状态信息数据通过集成的故障诊断系统对设备数据进行分析，然后将得出的结果传输给决策支持系统，进行自动决策管理。通过对设备日常的状态进行监测，随时对比历史数据进行分析，这样就可以在设备还未正式出现故障时进行预警，以及安排合理的设备维护计划，这样很大程度地减少了设备的停机时间，减少损失和设备的维修成本。

# 设备管理系统的设计和主要的实现步骤

## 现有的设备管理系统

1、风塔设备管理系统：本系统围绕设备的’进、出、维、修、检’各个环节进行科学管理,提供针对设备的“采购 -- 验收 -- 入账 -- 养护 --维修-- 盘点 -- 报废” 全生命周期跟踪管理，以预防性维护及预测性维修为中心，帮助企业实现设备的全面信息化管理，降低设备故障率，保持设备稳定性，实现企业资产效益的全面提升[3]。它主要包括的系统功能有：设备采购、设备处置、设备管控、设备维保、设备报修、备品备件。其优点是通用性强，可运用于多种场景，如各类工厂、实验室、机关、学校、企业等单位；缺点是没有其独有的特点，只是满足了一个设备管理系统的基本需求。

2、OA设备管理系统：将系统智能化，可通过智能语音交互，消除软件界面；它与OA平台以及其他异构系统集成构建统一工作平台，不止是单一的系统；将流程全部转为系统，全程电子化。其优点是系统智能化，且与其他系统融为一体，作为一个企业的设备管理系统非常有效，缺点是必须和OA平台统一构建，没有其他系统的灵活性。

3、乾元坤和智能设备管理系统平台：经过10年的迭代发展，已趋于成熟完善，主要包含设备资产管理、设备状态管理、设备采购管理、设备计划管理、设备处理管理、设备运行管理、设备运行监控、设备调拨管理、设备变动管理、设备维修管理、设备报废管理、统计分析查询等十二大模块。其优点是功能非常详细完善，对设备的整个生命周期进行全面管理，可扩展性强。

## 设备管理系统的实现

首先设备管理需要遵循的原则包括：

设备综合管理原则[4]——贯彻“全员、全过程、全效率”的设备综合管理思想，加强全过程管理，利用系统管理思想，追求设备管理综合效率最高。

设备状态预知维修原则[4]——通过推行设备状态预知维修管理模式，积极开展日常点检、专业点检、和重点设备的精密点检，把定性判断与定量分析有机结合，技术掌握设备技术状态，实施状态预知维修，实现设备管理由静态管理项动静管理结合的转变。

设备维修持续改进原则[4]——管理过程中对设备进行有效升级，形成长期规范化科学管理。实际上需要做定期分析评估，实施持续改进，通过设备状态预知维修模式管理为重点构建设备管理长效机制使设备管理水平不断提升。

其次设备管理的的内容包括：

设备的选择和评估：根据技术上先进、经济上合理的原则和生产上需要，正确的选择设备，同时要进行技术经济论证和评价，以选择最佳方案。

设备的使用：正确、合理地使用设备、安排生产任务，可以减轻设备的亏损、延长使用寿命，防止设备和入身事故；减少和避免设备闲置，提高设备利用率等。

设备的检查、维护保养和修理：设备缺陷及事故管理、预防性维修、维修计划排程[7]。

设备的改造和更新：根据生产经营的规模、产品品种、质量和发展新产品、改造老产品的需要，有计划、有重点的对现有设备进行改造和更新。

设备日常管理：主要包括设备的分类、登记、编号、封存、报废、事故处理和技术资料管理等。

然后设备管理系统可采用的技术：

SSM架构技术：SpringMVC框架是Spring FrameWork所推出的后续产品，是Spring框架基于MVC设计模型的用于构建Web应用程序的一个模块[5]。

基于jQuery的HTML5语言：HTML5是构建Web内容的一种语言描述方式，也被认为是互联网的核心技术之一；jQuery是一个快速、简洁的JavaScript框架，是一种优秀的JavaScript代码库[9]。

B/S架构：B/S模式应用系统由浏览器和服务器组成[6]。在这种架构下，用户工作界面是通过WWW浏览器来实现，极少部分事务逻辑在前端（Browser）实现，但是主要事务逻辑在服务器端(Server)实现，形成所谓三层3-tier结构[7]。

Vue.js：近年来用户数量增长最快的前端开发框架。目前，GitHub网站上已经达到了156000颗星，位居Angular、React和Vue.js前端3大框架之首[8]。

# 实现设备管理系统要解决的关键问题

1、当前企业设备管理存在的问题是什么?

第一，对设备管理缺乏正确的认知[4]，随着市场竞争愈加激烈，许多企业将重心放在体制机制的改革更新以及新技术的推广，忽略了设备管理的重要性，弱化设备管理的地位。第二，设备管理缺乏科学的机制[4，16]，在企业发展建设的过程中，企业会根据市场需求对企业内部机构进行调整，但是有一些企业在设备管理方面的调整未做到准确，导致整体缺乏了科学专业性的规范化管理。第三，管理人员的技术水平不够[10]，在某些单位，设备管理人员属于后勤地位，并不是专业的人员，导致技术能力和服务意识滞后，管理人员没有钻研设备管理和研究电子化办公设备的意识，所以技术水平也不会有提升。

2、怎样提高设备管理水平?

（1）实施设备管理责任制：明确设备管理分管领导和职能部门；（2）创建设备状态预知维修模式：配置必要的设备点检测仪器和专职的点检人员，定时开展设备点检，做好设备故障和事故管理；（3）积极探索科学检修机制，为设备运行创造良好环境[14]：采用PDCA[12]循环持续设备质量改进，将设备的检修分为四个阶段：计划、执行、检查、处理；（4）加强队伍建设：管理人员的专业素质直接影响设备的管理水平[11]，所以人员的培养在设备管理上也很重要。

# 总结

本文总结了设备管理的实际意义和影响，以及设备管理系统实现的大概步骤和需要注意的问题，分析了当前设备管理在国内外的研究现状。随着社会的发展进步，各个企业、高校、实验室、工厂都离不开设备，所以设备监控管理系统的应用也会越来越广泛，随着计算机技术、网络技术的发展，设备管理系统也会越来越智能化以及简洁化。
