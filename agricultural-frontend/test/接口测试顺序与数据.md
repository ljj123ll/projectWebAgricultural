## 一、测试环境配置

### 1.1 基础信息

- **Content-Type**: `application/json`
- **认证方式**: JWT Token (Bearer)

## 五、注意事项

1. **Token管理**: 登录成功后保存返回的 token，后续请求需要在 Header 中携带 `Authorization: Bearer {token}`

2. **短信验证码**: 测试环境可使用固定验证码 `123456`，生产环境需要真实发送

3. **订单状态流转**:
   - 待付款(1) → 待发货(2) → 待收货(3) → 已完成(4)
   - 待付款(1) → 已取消(5)（用户取消/超时未付款）
   - 支付异常(6) 作为异常状态记录

4. **售后状态流转**:
   - 待商家处理(1) → 协商中(2) → 已解决(3)
   - 待商家处理(1) → 管理员介入(4) → 已解决(3)
   - 已驳回(5) 作为商家拒绝后的终态

5. **审核状态**:
   - 商家审核: 0-待审核 1-通过 2-驳回
   - 商品审核: 0-待审核 1-已上架 2-已下架 3-已驳回
