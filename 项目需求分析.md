# 一、用户端
（核心定位：面向普通消费者，兼顾农村用户易用性，轻量化交易 + 助农信息浏览）

用户端围绕 “商品浏览 - 下单交易 - 售后评价 - 助农信息获取” 核心链路设计，所有功能以 “简单、直观、适配移动端” 为原则，具体模块及功能如下：
1. 登录与注册模块
注册：仅需手机号 + 短信验证码完成注册，无需复杂用户名 / 密码设置（可选补充昵称，默认手机号脱敏显示）；
登录：支持「验证码免密登录」（核心）、账号密码登录（备用），登录态通过 Redis 缓存，支持自动登录（7 天有效期）；
找回密码：仅需手机号验证码重置，无额外验证步骤，适配农村用户操作习惯。
2. 首页与商品浏览模块
首页布局：极简设计，包含 “推荐商品、助农热销榜、滞销农产品专区（可选）、助农新闻入口” 四大核心板块，按钮 / 字体放大适配移动端；
商品搜索：支持商品名称、商家名称模糊搜索，搜索结果支持 “按评分（高→低）、销量（高→低）、价格（低→高 / 高→低）” 排序；
商品筛选：可按 “产地（四川市县乡镇下拉选择）、商品品类（生鲜 / 干货 / 粮油等）” 筛选，快速定位目标商品；
库存提示：商品列表页标注 “库存仅剩 X 件”，避免用户无效浏览。
3. 商品详情与溯源模块
基础信息展示：商品图片（多图轮播）、名称、价格、品类、库存、详细介绍（种植 / 保存 / 运输简述）；
商家入口：可点击进入商家主页（查看商家资质、全部商品、商家简介）；
溯源功能：页面显著位置展示溯源二维码，支持 “网页点击查看 H5 溯源信息” 或 “扫码查看”，溯源信息包含：种植周期、产地、施肥类型、保存方式、运输方式（商家填写的溯源数据）；
加购操作：选择商品种类 / 数量（实时校验库存，库存不足时弹窗提示 “当前仅 X 件可购”），加入购物车。
4. 购物车与订单交易模块
购物车管理：查看已加购商品（显示名称、价格、数量、库存），支持 “修改数量（校验库存）、删除商品、清空购物车、结算”；
订单生成：点击结算后，选择收货地址（默认地址优先），生成待付款订单，系统自动扣减库存；
支付规则：订单生成后 15 分钟内未付款自动取消，取消后库存自动回滚；支付页面支持 “支付宝 / 微信支付”，标注支付金额、订单编号；
支付异常处理：支付中 / 支付失败时，订单状态标注 “支付异常”，提示用户 “重新支付”，避免重复下单，先使用模拟支付；
物流跟踪：订单发货后，仅展示物流号，物流单号复制（先完成初级内容）。
5. 个人中心模块
个人信息管理：修改昵称、密码、头像（支持手机拍照上传），收货地址管理（新增 / 编辑 / 删除 / 设为默认，地址选择省 - 市 - 县 - 乡镇下拉框，避免手动输入错误）；
订单管理：
全部订单：按 “待付款、待发货、待收货、已完成、已取消” 分类展示；
待付款：可重新支付 / 手动取消订单；
待发货：查看订单进度，联系商家（留言）；
待收货：确认收货 / 申请售后；
已完成：查看订单详情、评价商品；
已取消：显示取消原因（超时未付款 / 用户手动取消 / 商家取消）；
评价管理：查看已发布的商品评价。

6. 售后与评价模块
评价功能：订单签收后可评价商品，支持 “文字 + 图片 / 视频”、5 分制打分，评价提交后同步至商品详情页，作为商品排序依据；
售后功能：
发起售后：订单页 “申请售后” 按钮，选择售后类型（退款 / 退货 / 换货），填写原因（下拉选择：变质、少发、规格不符等）+ 上传凭证图片，提交后通知商家；
售后沟通：通过订单留言与商家沟通，商家回复后弹窗提醒；
管理员介入：协商无果时，可点击 “申请管理员介入”，等待平台判定。
7. 助农新闻板块
新闻展示：按 “农业政策、扶贫案例、产地动态” 分类展示 MD 格式新闻，内容自适应移动端（字体放大、图片适配）；
新闻操作：支持新闻详情查看，无冗余功能。

# 二、商家端
（核心定位：面向农户 / 合作社，极致简化操作，聚焦 “商品管理 - 订单处理 - 收益查看”）
商家端所有功能以 “降低操作门槛、适配农村用户” 为核心，仅保留高频核心功能，避免复杂交互，具体模块及功能如下：
1. 登录注册与店铺审核模块
注册：手机号 + 验证码注册，需填写 “农户 / 合作社名称、联系人、联系电话”，提交后进入店铺审核环节；
登录：支持验证码免密登录 / 账号密码登录，登录后首页展示 “待办事项（未处理订单 / 售后）、7 天核心数据”；
店铺审核进度：实时显示 “审核中 / 审核通过 / 审核驳回”，驳回时标注具体原因（如 “资质图片模糊、缺少农户身份证明”）。
2. 店铺信息管理模块
基础信息维护：修改店铺名称、简介、联系方式，上传 / 更新资质图片（农户身份证明 / 合作社营业执照）、收款账户（修改需管理员审核，支持 1 分钱账户验证）；
收款账户管理：查看账户状态（已验证 / 未验证），对账明细入口（查看每笔订单收入、打款记录）。
3. 商品管理模块（含溯源功能）
商品操作：新增 / 编辑 / 上下架 / 删除商品，核心字段简化为：
基础信息：商品名称、品类（下拉选择）、价格、库存（数字输入框）；
多媒体：商品图片（批量上传、自动压缩 / 裁剪，多图排序）；
详细信息：种植信息（种植周期 / 产地 / 施肥类型）、仓库存储信息、运输信息（运输方式下拉选择）；
溯源二维码：填写完溯源信息后，系统自动生成二维码，支持 “预览、下载图片（用于打印贴包装）”；
库存管理：修改库存（实时同步至用户端），设置库存预警值（如 10 件），库存低于预警值时弹窗 + 短信提醒；
新手引导：15 秒短视频引导（如 “如何上传商品、如何生成溯源二维码”），点击即可查看。
4. 订单管理模块
订单列表：按 “待售后、待发货、待收货、已完成、已取消” 分类，新订单置顶显示并标注红点提醒；
订单操作：
待处理：查看订单详情（商品、数量、用户地址、金额），取消订单（需选择取消原因：库存不足 / 用户取消等）；
待发货：填写物流信息（快递公司下拉选择政府谈好的物流商、物流单号（格式校验）），支持 1 次物流单号修改，填写后用户端同步 “已发货” 状态；
已完成：查看订单最终状态，确认打款进度；
订单提醒：新订单 / 订单状态变更时，触发短信提醒（如 “您有 1 笔新订单，请及时处理”）。
5. 售后处理模块
售后列表：按 “待处理、协商中、已解决、管理员介入” 分类，未读售后标注红点；
售后操作：查看用户售后申请（原因、凭证），通过订单留言回复 / 协商，同意 / 拒绝退款（拒绝需填写原因）；
留言提醒：未读用户留言弹窗，避免错过沟通。
6. 数据统计与消息提醒模块
数据统计：
核心数据：近 7 天销量、营收（直观柱状图 / 数字展示）；
热销商品：TOP3 热销商品（名称 + 销量），帮助农户调整库存 / 定价；
消息中心：查看系统通知（审核结果、库存预警、打款提醒）、用户留言、售后通知；
短信提醒：覆盖新订单、售后留言、库存预警、审核结果、打款失败等关键场景。
7. 对账与补贴模块
对账明细：查看每笔订单的 “订单金额、实际营收、打款时间、打款状态”；
补贴明细：查看政府物流补贴金额，按照每笔订单补贴运费，直接打款到户。

# 三、管理员端
（核心定位：平台 / 政府管控，权限拆分，聚焦 “审核 - 风控 - 资金 - 运维”）
管理员端通过隐藏接口登录，强调 “权限管控、风险可控、数据闭环”，具体模块及功能如下：
1. 安全登录模块
登录验证：隐藏接口入口 + 账号密码 + 短信验证码，双重验证保障安全；
角色权限：拆分两类角色，权限隔离：
超级管理员：全功能权限；
内容管理员：仅商家 / 商品 / 评论审核，新闻增删改查；
2. 用户管理模块
用户列表：查看所有注册用户（脱敏显示手机号 / 地址），支持按注册时间筛选；
用户操作：新增（仅后台创建测试用户）、编辑（修改用户状态：正常 / 禁用禁止登录）、删除、查看用户订单记录；
3. 商家与店铺审核模块
商家审核列表：查看待审核 / 已审核 / 已驳回商家，支持批量审核；
审核操作：查看商家资质图片、基本信息，审核通过 / 驳回（标注驳回原因）；
商家管理：查看所有审核通过商家，禁用违规商家（禁用后店铺商品自动下架），查看商家对账 / 打款记录。
4. 商品审核模块
商品审核列表：查看商家提交的待审核商品，审核是否存在虚假宣传（如标注 “有机” 无凭证）、图片违规，全部商品，待审核，已审核；
审核操作：审核通过 / 驳回（标注原因）。
5. 订单与资金管控模块
订单管理：
查看全平台订单，按 状态 / 时间筛选，查看异常订单（同一地址大额多单、刷单订单），手动取消；
物流管控：查看物流超时订单（超 5天未送达），提醒商家核实，标注物流异常原因；
资金管控：
自动打款：订单用户确认收货后 7 天（无售后纠纷），脚本自动打款至商家收款账户，打款失败后重试 3 次，失败则提醒管理员手动处理；
退款处理：用户申请退款时，未打款则拦截打款，已打款则从商家后续营收扣回，记录退款台账；
打款台账：查看所有打款记录（成功 / 失败），导出打款明细（财务对账）；
6. 评论与内容管理模块
评论管理：
查看所有用户评价，支持按商品 / 商家筛选；
风控过滤：管理员手动审核待审核评论；
评论操作：删除违规评论；
新闻管理：对 MD 格式助农新闻进行增删改查，按分类（农业政策 / 扶贫案例 / 产地动态）管理，预览发布效果。
7. 系统运维与数据看板模块
系统日志：记录所有管理员操作（增删改查），支持按时间 / 操作类型查询，方便审计溯源；
数据备份：每日自动备份订单 / 库存 / 商家核心数据，支持手动备份 / 恢复；
平台数据看板：展示全平台核心数据：总订单量、总营收、各商家销量排行（只展示前几名），支持按周 / 月筛选；
8. 助农特色管控模块
滞销专区管理：审核商家的滞销农产品申请，将符合条件的商品加入滞销专区，设置优先展示；


技术路线：本系统基于B/S架构设计，采用前后端分离模式
运行环境：Windows 11；
开发工具：VS Code、IntelliJ IDEA、Navicat、Apifox、Git、Maven；
主要开发语言：TypeScript、Java、HTML5、CSS3；
前端框架：Vue 3、Vite、Pinia、Element Plus；
后端技术：SpringBoot、MyBatis-Plus、Lombok、JWT、Swagger；
数据库：MySQL、Redis。

为了提升完善度，建议分阶段优化：短期修复高严重度问题，中期补充功能，长期增强扩展性。具体如下：
1.短期（安全与核心功能）：
1.集成真实支付SDK（支付宝/微信），添加HTTPS和数据加密（AES）。
2.增强溯源验证：对接政府API或使用区块链（如Hyperledger）存储不可篡改数据。
3.添加用户反馈模块和多支付选项；补贴模块增加审核流程。
2.中期（用户体验与扩展）：
1.用户端：引入推荐算法（基于Pinia状态或后端ML模块）。
2.商家端：支持批量操作和实时聊天（WebSocket via Spring）。
3.管理员端：添加行为分析（集成ELK日志栈）和自动化风控规则。
4.优化性能：Redis扩展到库存，使用ShardingSphere分库。
3.长期（运维与合规）：
1.引入Docker/Kubernetes部署，支持云环境（如阿里云）。
2.添加自动化测试和监控工具；合规上补充隐私政策页面。
3.进行用户测试（农村用户调研），迭代UI/UX。
